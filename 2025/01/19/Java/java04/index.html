# 資料型態

上一篇我們有提到變數有哪些型態，而其中又有一些小細則，在這篇會稍微談一下

# 整數型態

整數的資料預設會是 int 型態

因為 20000000000 預設為 int 型態，他會覺得你宣告了一個超過 int 範圍的變數
error: integer number too large

那我們要怎麼宣告 long 型態的整數呢？
做法是在整數後面加上大寫的 L

# 浮點數型態

與整數的概念類似，浮點數的資料預設是 double 型態


那我們要怎麼宣告 float 型態的浮點數呢？
做法是在整數後面加上大寫的 F

這邊輸出 m 會發現結果是 3.1415927，這是因為 float 為單精度浮點數，範圍只到小數點後七位

接著來看看在 java 中要怎麼進行資料型態的轉換

# 資料大小

上一篇有提到各種資料型態的大小
其中浮點數型態的範圍皆大於整數型態

double>float>long>int>short>byte

而這個大小在型態轉換中非常重要

# 小範圍轉換成大範圍

在 java 中，小範圍轉換成大範圍的型態時會自動轉換

第三行程式碼隱含了由 int 型態轉換成 long 型態的概念，前面提到的因為預設為 int 而報錯的例子是因為超過了 int 型態的範圍

# 大範圍轉換成小範圍

在 java 中， 大範圍轉換成小範圍的型態時會產生錯誤

第五行程式碼隱含了由 double 型態轉換成 float 型態的概念，因為 10.5 被預設為 double 型態，從 double 型態轉成 float，由大範圍轉換成小範圍的型態會報錯

# 大範圍轉成小範圍：強制轉換

在 java 中， 大範圍轉換成小範圍的型態可以使用 (資料型態) 強制轉換
